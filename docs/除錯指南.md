# 除錯指南：先找第一個錯誤，不要同時改五件事

## 1. Vivado 找不到 module

1. 在 Sources 視窗確認檔案真的加入目前 project。
2. 檢查 module 名與 instantiation 名。
3. 檢查 `.sv` 是否被當成 SystemVerilog；必要時看 File Type。
4. 查看編譯順序，package／被引用 module 應先編譯。
5. 不要把 solution 與 starter 同時加入，否則會 duplicate module。

## 2. TB 一開始全部是 X

- reset 初值是否在 time 0 定義；
- synchronous reset 是否至少跨過一個 rising edge；
- TB 是否有兩個 initial/always 同時驅動同一 signal；
- DUT register 是否在 reset branch 全部賦值；
- array memory 未初始化是否正被讀取。

## 3. Valid/ready 少資料或重複資料

在 TB 建兩個 counter：

```text
input_count  += (s_valid && s_ready)
output_count += (m_valid && m_ready)
```

所有 queue push/pop 都只以 transfer condition 為準。不要用 valid 的 rising edge 當 transfer；valid 可以連續維持多 cycle。

## 4. INT4 負數錯誤

1. 用 `4'hF=-1`、`4'h8=-8` 做最小測試。
2. 先把 nibble 指派到 `logic signed [3:0]`，再 sign-extend。
3. 對 part-select 明確 `$signed(...)`；part-select 常失去 signed 屬性。
4. Python unpack 時若 nibble >= 8，要減 16。

## 5. 2:4 mask 對不上

列印一組人工資料，不先用 random：

```text
lanes:   0   1   2   3
act:    10  20  30  40
mask:    0   1   0   1
weight:      -3      +5
expected = 20*(-3) + 40*5 = 140
```

同時在 Python、TB 與 RTL 註解寫出 bit 0 對 lane 0。若 encoding 用 index，測 index 排序與重複 index 的非法情形。

## 6. XDC 報錯

- `get_ports` no objects：XDC port 名與 top 不一致。
- unconstrained clock：`create_clock` 沒套到 top clock，或 AXI 設計應由 generated clock／PS FCLK 約束。
- NSTD/UCIO：有 top I/O 未指定 IOSTANDARD/PACKAGE_PIN；若只是內部測試 port，應改 top，而不是隨意分配腳位。
- 不要用 `set_property SEVERITY Warning` 壓掉真正的腳位錯誤。

## 7. Block Design Validate Design 失敗

從第一個 Critical Warning 開始：

1. AXI clock association；
2. reset polarity；
3. 未接的 AXI channel；
4. address segment；
5. DMA stream data width；
6. TLAST 是否存在。

Run Connection Automation 後仍人工追一遍資料路徑與 clock/reset。

## 8. DMA wait 卡住

依序確認：

1. S2MM receive transfer 是否在 send 前啟動。
2. accelerator 最後一筆是否送 `TLAST=1`。
3. output `TVALID` 是否真的與 DMA `TREADY` 握手。
4. AXI DMA 是否同時啟用 MM2S/S2MM。
5. buffer dtype 與 stream width 是否一致。
6. DMA maximum transfer length 是否足夠。
7. bit/hwh 是否為同一次 build 且同 basename。

## 9. Timing fail

不要先降低 clock 假裝成功。先看 critical path，確認是否可用 pipeline 切開；更新 latency 後同步修改 TB。若最後為了板上 demo 降 clock，報告要同時保留原目標與實際達成值。

## 10. 問問題時要附什麼

為了讓別人能幫你定位，附上：

- Lab 與檔名；
- Vivado/PYNQ 版本與 part；
- 第一個 error（不是最後 100 行）；
- 重現步驟；
- 預期與實際；
- 最小測試向量；
- relevant waveform 或 console；
- 你已排除的項目。
